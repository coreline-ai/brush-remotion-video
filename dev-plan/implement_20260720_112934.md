# implement_20260720_112934.md

작성 일시: `2026-07-20 11:29:34 KST`

이 문서는 기존 영상 제작 스킬의 자동 BGM 선택에 Stable Audio 3 MLX 피아노 후보를 연결하고, 스킬별 적용 범위와 최종 승인 경계를 고정하기 위한 작업 문서다.

## 개발 목적

기존 `bin/build.py` 기반 영상 스킬에서 BGM이 필요한 경우 영상 분위기·프로파일·포맷에 맞는 피아노 BGM을 자동으로 우선 생성한다. Stable Audio 런타임이 없거나 120초 제한을 넘는 경우에는 기존 검증 catalog/synth 경로를 안전하게 fallback하고, Stable Audio 후보는 사람 청취 승인 전 최종 납품에 사용하지 않는다.

## 개발 범위

- `bgm.mode: piano-auto` 요청 계약과 프로젝트 무드/프로파일 기반 prompt 조립
- `bin/build.py` 공용 `stage_mix()`의 Stable Audio 우선 자동 선택
- 무음/ambient 자동 BGM과 내레이션/TTS 명시적 `piano-auto`의 구분
- Stable Audio 후보 산출물·manifest·승인 상태를 기존 build/audit/final gate와 연결
- 기존 `brush-video` 계열 프로파일 스킬 문서의 적용 여부·사용법 동기화
- 각 스킬별 적용/비적용 판단표와 회귀 테스트

## 제외 범위

## 결정 기록

- 공용 `bin/build.py`를 사용하는 7개 영상 스킬은 코드 중복 없이 한 경로로 통합한다.
- `bgm` 블록이 없고 음성/TTS가 없는 영상은 `piano-auto`를 1순위로 시도한다.
- 음성/TTS/Whisper 영상은 말소리 명료도를 위해 자동 생성하지 않는다. 사용자가 `bgm.mode: piano-auto`를 명시하면 자동 덕킹과 함께 생성한다.
- Stable Audio는 `STABLE_AUDIO_3_MLX_ROOT`가 있고 총 길이 15~120초일 때만 사용한다. 미설치·장편·생성 실패는 기존 catalog, 마지막으로 synth로 fallback한다.
- `piano-auto` 후보는 `PENDING_USER_LISTENING` 상태로 생성되며 일반 preview/audit에는 사용 가능하지만 `--final`은 명시적 승인 manifest 없이는 거부한다.
- 승인 후보를 기존 `assets/bgm/catalog.json`에 자동 등록하지 않는다. 프로젝트별 generated manifest를 사용한다.

## 공통 진행 규칙

- 각 Phase는 앞선 Phase의 자체 테스트 완료 후에만 시작한다.
- 구현 중 발생한 이슈는 해당 Phase에서 수정하고 기록한다.
- 기존 `bgm.mode: off|synth|asset|playlist`의 동작과 기존 프로젝트를 보존한다.
- 모델 미설치 환경에서도 기존 영상 build가 실패하지 않도록 implicit auto만 fallback한다. 명시 `piano-auto`는 원인 있는 오류를 낸다.
- 기존 사용자 변경·오디오·golden/output을 reset, restore, 대량 포맷하지 않는다.
- 새 공통화는 `bin/build.py`가 공유하는 최소 bridge와 기존 Stable Audio adapter 재사용으로 제한한다.
