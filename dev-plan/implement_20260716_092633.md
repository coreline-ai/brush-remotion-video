### project.yaml 계약

MeloTTS 예시:

```yaml
input:
  script: narration.txt
  tts:
    engine: melo-ko
    voice: kr-default
    language: ko
    speed: 1.00
    pauseMs: 350
    timing: tts
```

Qwen3-TTS 예시:

```yaml
input:
  script: narration.txt
  tts:
    engine: qwen3-base
    voice: f1-reference
    language: ko
    reference:
      audio: inputs/voices/f1-reference.wav
      transcript: inputs/voices/f1-reference.txt
    speed: 1.00
    pauseMs: 350
    timing: tts
```

- `reference.audio`와 `reference.transcript`는 `project.yaml` 파일 기준 상대 경로로 해석한다.
- `input.tts`의 허용 필드는 `engine`, `voice`, `language`, `speed`, `pauseMs`, `timing`, `reference`이며 엔진별 미지원 필드와 unknown field는 hard fail한다.
- 이 계획의 신규 엔진 언어는 `ko` 하나이며, 생략 시 `ko`로 해석한다. `ko`는 Melo의 `KR`, Qwen의 `Korean`으로 매핑한다.
- `speed`는 bool·문자열·NaN·infinity가 아닌 유한 숫자 `0.70~2.00`만 허용한다. `pauseMs`는 bool·문자열·실수가 아닌 0 이상 정수로만 받으며 조용한 형변환을 하지 않는다.
- `reference.audio`와 `reference.transcript`는 YAML 부모 기준으로 canonicalize한다. regular file·읽기 권한·SHA-256을 확인한 뒤 `data/{projectId}/tts/.work/{requestId}/reference/`에 원자적으로 복사하고 다시 hash를 검증한다. worker에는 이 controlled copy만 전달하며 성공·실패·취소 후 staged copy를 삭제한다.
- manifest와 일반 로그에는 외부 reference 절대 경로와 transcript 본문을 남기지 않고 voice ID, staged source kind, basename, hash만 기록한다.
- Qwen Base는 reference transcript가 없는 상태를 기본 허용하지 않는다. x-vector-only 모드는 이번 범위에 포함하지 않는다.
- 모든 엔진 출력은 adapter가 반환한 실제 sample rate를 기준으로 공통 계층에서 44.1kHz mono로 변환한 후 실제 샘플 길이를 타이밍 시계로 사용한다. 특정 엔진 native rate를 공통 계약으로 고정하지 않는다.
- `input.audio`가 있으면 항상 실더빙 branch를 우선한다. SRT가 있으면 `narration`, SRT가 없으면 `whisper`로 판정하고, 함께 있는 TTS/script는 경고 후 실행하지 않는다.
- audio가 없을 때 `script + srt + tts`를 함께 주면 텍스트 source가 모호하므로 hard fail한다. `script + tts`는 script, `srt + tts`는 SRT 본문을 사용한다.
- `timing: tts`는 신규 엔진의 유일한 실행 모드다. 기존 Supertonic의 `timing: srt`는 입력 호환을 위해 받되 SRT를 텍스트 source로만 사용하고 합성 길이로 타이밍을 다시 만들며, manifest에 `requestedTiming=srt`, `appliedTiming=tts`와 경고를 기록한다.
- `tts-bench` runner의 single-shot/`--stdin` JSONL 계약을 대상 protocol의 baseline으로 삼고, 대상 worker의 batch·prompt reuse는 별도 확장으로 명시한다.
- `tts-bench` Qwen registry는 autoregressive Base cloning이 긴 prompt에서 600초 cell timeout을 일으켜 비활성화한 사례가 있다. 대상 구현은 `STARTUP_TIMEOUT`·`GENERATION_TIMEOUT`으로 구분하며, timeout을 `ENVIRONMENT-BLOCKED`, 무한 대기, 다른 voice fallback으로 숨기지 않는다.
- `tts-bench`의 샘플 WAV는 비교·검증용 local fixture로만 사용한다. fixture가 없는 환경에서는 자동으로 다른 reference를 선택하지 않고 설치/fixture 차단으로 기록한다.

### 합성·정규화 처리 순서

1. YAML 타입·엔진별 필드·입력 우선순위와 reference pair를 검증한다.
2. engine·package·pinned model snapshot을 resolve하고 normal build에서는 network를 사용하지 않는다.
3. adapter가 문장별 native float samples와 실제 sample rate를 반환한다. Melo speed는 engine에서 적용하고 Qwen speed는 합성 뒤 공통 `atempo`에서 적용한다.
4. speed 적용 뒤 mono shape·finite·범위를 검사하고, `scipy`로 44.1kHz resample한 뒤 같은 검사를 반복한다. `1e-6` 이내 부동소수점 초과만 명시적으로 clamp·계수 기록하며 그보다 큰 초과는 hard fail한다.
5. 정규화된 문장 사이에 `pauseMs` 무음을 붙이고 postprocess 이후 실제 sample 수로 SRT를 만든다.
6. WAV·SRT·manifest를 임시 파일에 쓴 뒤 검증하고 원자적으로 교체한 후에만 `stt` ledger를 완료 처리한다.

### 모델 snapshot·네트워크 정책

- `scripts/tts-doctor.py --prepare <engine>`만 network와 모델 다운로드를 사용할 수 있다. 정상 `bin/build.py`는 local-only로 실행하며 다운로드나 network retry를 하지 않는다.
- Melo는 pinned snapshot의 `config.json`·`checkpoint.pth`를 `TTS(config_path=..., ckpt_path=...)`에 직접 전달한다.
- Qwen은 pinned snapshot 디렉터리를 `from_pretrained()`에 전달하고 local-only 환경에서 로드한다.
- snapshot 누락은 `MODEL_MISSING`으로 hard fail하고 doctor 준비 명령을 안내한다. 최신 revision으로 자동 전환하거나 다른 voice/model로 fallback하지 않는다.

### Qwen worker protocol v1

- worker는 빌드당 한 process로 시작하고 모델을 한 번 로드한 뒤 `{"ready":true,"protocolVersion":1,"modelRevision":"fd4b254389122332181a7c3db7f27e918eec64e3"}` JSONL을 보낸다. startup timeout은 300초다.
- orchestrator는 ordered `sentences[{id,text}]`, language, staged reference path/hash, model revision을 한 request로 보내고 worker는 voice clone prompt를 한 번 생성해 문장 순서대로 재사용한다. 모델 batch 크기는 내부 구현 세부로 두며 YAML 옵션으로 노출하지 않는다.
- generation timeout은 request당 600초다. 취소·timeout 시 5초 graceful 종료 후 process group을 강제 종료하고 controlled temp output을 삭제한다.
- error code는 `DEPENDENCY_MISSING`, `MODEL_MISSING`, `INVALID_REQUEST`, `REFERENCE_MISMATCH`, `STARTUP_TIMEOUT`, `GENERATION_TIMEOUT`, `OOM`, `CANCELLED`, `PROTOCOL_ERROR`, `MODEL_ERROR`로 고정한다. `ENVIRONMENT-BLOCKED`는 dependency/model 미설치 테스트 상태에만 사용하고 runtime timeout을 이 상태로 숨기지 않는다.
- `retryable=true`는 timeout 계열의 안내 metadata로만 사용하며 orchestrator가 자동 재시도하지 않는다. 나머지 오류는 설정·환경·입력을 수정한 뒤 명시적으로 재실행한다.
- response output은 request sentence ID와 일대일 대응하는 relative filename·sample rate·duration을 포함한다. ID 누락·중복·순서 불일치·절대 경로·stdout 비JSON 출력은 protocol failure다.

- Qwen 모델은 별도 Python 환경을 우선하며, 기본 pipeline 환경에 무거운 의존성을 강제로 합치지 않는다.

- 새 의존성은 공식 모델/패키지 설치 경로와 macOS·Python 3.11/3.12 실행 결과를 확인한 뒤 추가한다.
- 공개 배포 전에는 모델 라이선스, reference voice 권리, AI 합성 고지를 각각 확인한다.
- 외부 reference는 명시적인 읽기 전용 입력으로만 취급하고, 임의 외부 파일 쓰기·worker 출력 경로 탈출·reference 원본의 저장소 재배포는 금지한다.
- 검증한 reference는 controlled build directory에 stage한 뒤 재검증하며, 외부 절대 경로와 transcript 본문을 로그·manifest에 남기지 않는다.
- worker stdout에는 protocol JSON만 기록하고 로그는 stderr로 보낸다. timeout 시 자식 process group을 5초 grace 후 종료하고, 취소 시 `SIGTERM`/필요한 `SIGKILL`과 controlled temp cleanup을 수행한다. idle cancel command는 `CANCELLED` JSON error 후 정상 종료한다.
- `tts-bench` reference runner의 JSONL 형식과 충돌하는 ad-hoc stdout 출력, 로그 혼입, 임의의 기본 reference·speaker fallback을 추가하지 않는다.
- cache signature에는 `ttsCacheSchemaVersion`, engine/model revision, adapter·normalizer version, language·voice·speed 적용 방식, pause, 대본·reference hash를 포함한다. signature 변경 시 `stt`부터 재실행한다.
- 모델 ID만 기록하지 않고 실제 사용한 pinned revision 또는 local model digest를 manifest와 cache에 남긴다.

- [x] 기존 YAML에서 `engine`을 생략하면 Supertonic 동작이 변하지 않는다.

## 잔여 리스크 / 후속 과제

- Qwen 1.7B CPU 추론 시간과 메모리 사용량은 호스트마다 달라질 수 있으므로 운영 기본값으로 자동 선택하지 않는다.
- `tts-bench`에서 확인된 Qwen Base 긴 prompt 600초 timeout 사례가 대상 host에서도 재현될 수 있으므로 startup/generation timeout을 분리하고 runtime timeout을 환경 차단으로 분류하지 않는다.
- MeloTTS `melotts_ko` 기준 환경은 `mecab-python3` 누락으로 import가 실패한 상태다. Phase 1에서 호환 버전을 pin하고 `pip check`·import·KR model load를 통과하기 전까지 환경 차단으로 유지한다.
- Qwen voice cloning 결과의 성별·억양 품질은 reference audio 품질과 권리 범위에 의존하며 자동 음질 판정으로 대체하지 않는다.
- Qwen non-1.0 속도 보정이 음질을 훼손하면 후속 작업에서 pitch-preserving time-stretch 정책을 별도로 검토한다.
- 공개 배포용 reference voice catalog와 동의 증빙은 본 계획의 구현 범위를 넘어 별도 정책으로 분리한다.
- `tts-bench/results`가 gitignore 대상이므로 local fixture가 없는 CI에서는 실제 음성 E2E가 아닌 명시적 환경 차단으로 남을 수 있다.
- 현재 shared file에 대규모 미커밋 변경이 있으므로 구현 도중 baseline hash가 달라지면 자동 병합하지 않고 소유 단계별로 재검토한다.
