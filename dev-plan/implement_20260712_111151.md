## 개발 목적

현재의 단일 합성 피아노 BGM을 유지 가능한 폴백으로 보존하면서, **로컬에 내려받은 상업적 사용 가능 BGM을 공통 오디오 엔진에서 재사용**할 수 있도록 한다.

- 공통 BGM 엔진과 라이선스 정책을 `brush-video` 본선에 구현한다.
- `pen-brush-video`를 포함한 기존 영상 스킬은 코드 사본 없이 구체적인 BGM 프리셋만 선택한다.
- 음원은 렌더 중 인터넷에서 받지 않는다. 공식 곡 페이지에서 먼저 로컬 다운로드하고 라이선스 증빙과 함께 등록한 뒤 빌드한다.
- BGM과 내레이션/TTS를 별도 트랙으로 처리하고, 게인·페이드·덕킹·플레이리스트 크로스페이드를 공통 처리한다.
- 완성본은 무음·클리핑뿐 아니라 LUFS, True Peak, 페이드, 덕킹, 라이선스 매니페스트까지 자동 검수한다.
- 기존 앰비언트 영상의 약한 음량을 실측 기준 약 `+4~6dB` 개선하되, 단순 진폭 증폭이 아니라 정규화와 리미터를 적용한다.

## 개발 범위

- **로컬 음원 다운로드·등록**
  - 선정된 Pixabay 음원 10개를 각 공식 곡 페이지에서 로컬로 다운로드한다.
  - MP3, 곡 URL, 다운로드 날짜, 곡명, 작가명, 라이선스 PDF/스크린샷, 선택적 Content ID 인증서를 함께 보관한다.
  - 로컬 원본의 SHA-256, 재생 길이, 오디오 스트림 정보를 추출하여 카탈로그에 등록한다.
  - 다운로드·증빙·등록 진행 상태를 명령과 체크리스트로 확인할 수 있게 한다.
- **공통 BGM 계약**
  - `project.yaml`에 내레이션 `input.audio`와 독립적인 최상위 `bgm` 블록을 추가한다.
  - `off | synth | asset | playlist` 모드를 지원한다.
  - 기존 프로젝트에 `bgm`이 없으면 현재 동작을 보존한다: 앰비언트는 합성 BGM, 내레이션/TTS/whisper는 기존 음성만 사용한다.
- **라이선스·Content ID 게이트**
  - 빌드 시작 전 로컬 파일, 해시, 출처, 라이선스 증빙, Content ID 상태를 검사한다.
  - `not-displayed`는 `미등록 보장`으로 해석하지 않고 위험 경고와 확인 일시를 기록한다.
  - 최종 프로젝트 데이터에 사용한 음원의 라이선스 매니페스트를 남긴다.
- **오디오 처리**
  - BGM 기준 레벨을 `-23 LUFS`로 정규화한다.
  - 앰비언트는 기본 `+5dB` 트림으로 약 `-18 LUFS`를 목표로 한다.
  - 내레이션 영상은 BGM 기본 `+3dB`, 음성 구간 `-8dB` 덕킹으로 시작하고 실측 후 범위를 보정한다.
  - 페이드 인 `1.8초`, 페이드 아웃 `2.0초`, 트랙 크로스페이드 `3.0초`를 기본으로 한다.
  - 최종 True Peak는 `-1dBTP` 이하로 제한한다.
- **장시간 영상**
  - 명시적 2~3곡 플레이리스트를 영상 길이에 맞춰 트림·연결한다.
  - 10분 초과 단일곡은 허용하되 director와 검증기가 플레이리스트를 권장 경고한다.
- **스킬 반영**
  - 공통 정책은 `brush-video` reference 한 곳을 진실로 둔다.
  - `pen-brush-video` 기본 에셋은 `Piano Dreamcloud Meditation`으로 한다.
  - 전체 공통 앰비언트 기본 에셋은 `Gentle Piano Meditation`으로 한다.
  - director가 무드·포맷·길이에 따라 구체적인 `assetId` 또는 `playlist.assetIds`를 작성한다.

## 결정 기록

- **독립 BGM 스킬을 먼저 만들지 않는다.** `brush-video` 공통 엔진·정책을 구현하고 기존 스킬은 프리셋만 가진다.
- 음원 수집이 반복적인 별도 업무로 커질 때만 후속 범위에서 `bgm-curator` 스킬을 검토한다.
- 렌더는 완전 오프라인·결정적으로 동작해야 하므로 웹 다운로드와 빌드를 분리한다.
- 직접 다운로드 URL은 만료되거나 바뀔 수 있으므로 카탈로그에는 공식 곡 페이지 URL을 영구 출처로 저장한다.
- 다운로드는 공식 곡 페이지에서 수행하고, `bin/bgm-assets.py import`가 이미 받은 파일을 로컬 에셋 저장소로 원자적 복사한다.
- 프로젝트의 `input.audio`는 계속 내레이션·실더빙 입력만 뜻한다. BGM 파일을 `input.audio`에 넣지 않는다.
- V1은 `bgm.mode: auto`를 지원하지 않는다. director가 자동 판단하더라도 최종 YAML에는 구체적인 `assetId`를 기록해 재현성을 보장한다.
- `bgm.licensePolicy: strict`가 기본이다. 증빙 누락·해시 불일치·Content ID 상태 `unknown`은 렌더 전에 실패한다.
- `not-displayed`는 strict에서 사용 가능하되 경고를 남긴다. `registered`는 기본 거부하고 인증서와 명시적 승인 정책이 있을 때만 후속 검토한다.
- 90일을 초과한 라이선스/Content ID 확인 기록은 빌드를 막지 않고 WARN으로 처리하되, 공개 업로드 전 재확인을 요구한다.
- 현재 합성 BGM은 삭제하지 않고 `synth` 모드와 기존 프로젝트 호환 폴백으로 유지한다.
- 신규 Python 오디오 라이브러리는 추가하지 않고 현재 필수 도구인 `ffmpeg/ffprobe` 필터를 우선 사용한다.

## 로컬 에셋·라이선스 저장 계약

```text
assets/
  bgm/
    catalog.json                  # 추적: assetId, 제목, 출처, 태그, 상대 로컬 경로
    catalog.schema.json           # 추적: 카탈로그 검증 계약

local-assets/                     # 전체 Git 제외
  bgm/
    <asset-id>/
      original.mp3                # 공식 페이지에서 받은 원본
      asset.json                  # 로컬 해시·실측 길이·등록 일시
      evidence/
        source-page.png           # 곡 페이지와 당시 표시 상태
        license.pdf               # 당시 라이선스 페이지 저장본
        content-id-certificate.*  # 제공되는 경우에만

data/<project-id>/audio/          # 빌드 산출물, 기존 data 정책대로 Git 제외
  bgm-normalized.wav
  narration-normalized.wav
  master.wav
  mix-report.json

data/<project-id>/licenses/
  bgm-manifest.json               # 최종 영상에 실제 사용한 음원·해시·증빙 목록
```

- `catalog.json`에는 절대 경로를 저장하지 않는다.
- `local-assets/`를 `.gitignore`에 추가한다.
- 빌드 산출물에는 원본 MP3나 라이선스 PDF를 복제하지 않고, 해시와 로컬 증빙 상대 경로를 가진 매니페스트만 만든다.
- 다른 머신에서는 동일 assetId의 파일을 다시 공식 다운로드·import해야 한다.

각 항목의 `[ ]`는 다음 6개가 모두 완료된 경우에만 `[x]`로 바꾼다.

1. 공식 곡 페이지에서 전체 길이 청취 또는 주요 구간 청취.
2. 페이지에 표시된 곡명·작가명·길이·라이선스·Content ID 상태 재확인.

4. 곡 페이지 스크린샷과 라이선스 PDF/스크린샷 저장.
5. `bin/bgm-assets.py import`로 파일·증빙·메타데이터 등록.

## 공통 진행 규칙

- 각 Phase는 앞선 Phase의 자체 테스트 완료 후에만 시작한다.
- 구현 중 발생한 이슈는 해당 Phase에서 수정하고 기록한다.
- 체크박스 상태를 실제 진행 상태와 맞게 업데이트한다.
- 문서에 없는 범위 확장은 하지 않는다.
- 요구사항이 불명확하면 가능한 케이스를 나누고 확인 후 진행한다.
- 한 기능은 유지보수 가능한 최소 책임 단위로 나누고, 각 단위의 역할과 경계를 계획에 적는다.
- 계획 밖 옵션/확장/부가 기능을 만들기 위한 분리는 하지 않는다.
- 공통화는 계획 범위 안의 중복을 줄일 때만 수행하고 과도한 추상화는 피한다.
- 기존 프로젝트 API, ffmpeg/ffprobe, Python 표준 라이브러리를 우선 사용하고 새 의존성은 범위상 필요할 때만 포함한다.
- 기존 미완료·미커밋 작업을 덮어쓰거나 되돌리지 않는다.
- 음원과 라이선스 증빙은 `local-assets/` 밖으로 복사하거나 Git에 stage하지 않는다.
- 외부 음원은 공식 페이지·공식 다운로드만 사용하고, 등록 당시 페이지 상태를 증빙한다.
- 빌드는 인터넷 연결 없이 완주해야 하며, 웹 상태 재확인은 공개 업로드 전 별도 단계로 수행한다.
- `bgm` 변경만으로 영상 프레임을 다시 렌더하지 않도록 StageLedger 경계를 유지한다.
- 모든 오디오 명령은 shell 문자열이 아니라 인자 배열로 호출하고, 파일 경로를 그대로 삽입하지 않는다.
- 전체 렌더 전에 30초 저해상도/소형 fixture로 믹싱 수치를 검증한다.
- 최종 E2E 결과는 `video-auditor` PASS와 사람이 직접 청취한 결과가 모두 있어야 완료로 처리한다.

## 잔여 리스크 / 후속 과제

- Pixabay 페이지나 Content ID 상태는 등록 후 변경될 수 있으므로 공개 업로드 직전 사람이 다시 확인해야 한다.

- 외부 음원의 실제 마스터링 특성에 따라 기본 gain·ducking 수치는 Phase 0/7 청취에서 조정될 수 있다.

- 음원 웹 검색·신규 후보 선별이 반복 업무가 되면 별도 `bgm-curator` 스킬 개발 계획을 새로 만든다.
