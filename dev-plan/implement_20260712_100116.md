## 개발 목적

**`pen-brush-video`** — 하나의 컬러 일러스트를 동일 좌표의 외곽선·채색 레이어로 분리하고,
**펜으로 샤프한 외곽선을 완성한 뒤 브러시로 영역을 채색하는 2단계 영상 프로파일**을
`brush_remotion_video` 본선에 편입한다.

- 사용자 접점은 독립 스킬 `pen-brush-video`로 제공한다.
- 실행은 기존 `bin/build.py`와 Remotion 엔진을 공유한다.
- 스킬은 코드 사본 없는 얇은 프로파일 스킬로 유지한다.
- 기본 사용 계약은 `drawing.profile: pen-brush` 한 줄이다.
- 기존 `brush`, `pen`, `shorts` 프로파일은 동작과 골든 결과가 변하지 않아야 한다.

## 개발 범위

- **이미지 2레이어 분리**
  - 하나의 최종 컬러 이미지에서 `outline RGBA`, `color RGBA`, `content mask`를 동일 좌표로 파생한다.
  - 외곽선은 로컬 대비 기반 잉크 추출을 사용하고 Canny+dilation 팽창 방식은 금지한다.
  - 컬러 레이어는 원본의 샤프한 선을 포함하며, 완성 시 이 레이어가 최종 화면의 기준이 된다.
- **2종 routes 생성**
  - outline routes: 펜이 외곽선·구조선을 그리는 좁은 경로.
  - paint routes: 자동 매크로 존 안을 넓은 브러시 획으로 채우는 경로.
  - 데모에 사용한 여우/나무/자전거 고정 좌표는 제거한다.
- **다단계 렌더 계약**
  - `Scene`에 선택적 `drawingPhases` 2단계 계약을 추가한다.
  - 기존 `scene.routes` 단일 경로는 하위 호환으로 그대로 유지한다.
  - outline은 paint 아래가 아니라 위(z-index 상위)에 유지하되, 채색 완료 직전에 0으로 페이드하여
    최종 화면에는 color 레이어의 원본 선만 남긴다.
- **파이프라인 통합**
  - `project.yaml`에 `drawing.profile: pen-brush`와 최소 선택 옵션을 노출한다.
  - clean/routes/props/qa 스테이지가 2레이어·2routes를 처리한다.
  - 가로 1920×1080과 세로 1080×1920을 동일 로직으로 지원한다.
  - 내레이션이 있으면 펜→브러시 전환이 문장 중간에 발생하지 않도록 cue 경계로 스냅한다.
- **독립 스킬 편입**
  - `skill/pen-brush-video/SKILL.md` + 필요한 reference만 추가한다.
  - `bin/install-skills.sh`에 symlink 설치 항목을 추가한다.
  - brush-director가 “외곽선 먼저, 이후 채색” 요청을 `pen-brush`로 라우팅한다.
- **QA·문서 환류**
  - 선 두께, 색상 누출, phase 순서, 커서 겹침, coverage, 완료 팝업을 자동 검증한다.
  - README·schema·pipeline·background prompt의 현재 상태 문서를 갱신한다.

- 스킬명은 `pen-brush-video`, 프로파일 값은 `pen-brush`로 고정한다.
- **독립 스킬 + 공유 엔진**을 채택한다. 스킬 내부 실행 코드 사본은 0개여야 한다.
- `schemaVersion`은 1을 유지하고 선택 필드를 추가한다. 기존 props 마이그레이션을 요구하지 않는다.
- `drawingPhases`는 이번 범위에서 `outline`, `paint` 정확히 2개만 허용한다.
- 외곽선은 채색 완료 직전 페이드 아웃하고, 최종 선은 color 원본의 선만 사용한다.
- 최종 선 두께 기준은 원본 대비 ±5%이며 데모 실측 오차 0.104%를 기준 fixture로 보존한다.
- paint routes의 매크로 존은 이미지에서 자동 산출하며 캔버스 절대 좌표 하드코딩을 금지한다.
- 신규 의존성 없이 현재 Pillow/numpy/scipy/scikit-image/React/Remotion 범위에서 구현한다.
- 스킬 편입은 엔진·가로/세로 E2E·회귀 테스트가 모두 통과한 뒤 마지막 단계에서 수행한다.

## 잔여 리스크 / 후속 과제

- imagegen 결과가 외곽선/색면 계약을 지키지 않으면 자동 추출 품질이 낮을 수 있다. 입력 품질 게이트로 조기 중단하고
  프롬프트 재생성을 안내한다.
- 복잡하게 연결된 한 장면은 정규화 공간 폴백으로 분할하며 현재 합성·실물 fixture는 통과했다. 향후 이미지 유형이 늘면 fixture를 추가한다.
- 장편 다씬에서 phase당 routes 수가 증가해 렌더 메모리/속도가 악화될 수 있다. 1씬 기준 통과 후 10씬 성능 스모크를
  후속으로 검토하되 이번 완료 게이트에는 포함하지 않는다.
- 사진 입력, 실제 손/붓 이미지, source-video 움직임 이식, 임의 N-phase 편집은 별도 워크스트림으로 남긴다.
