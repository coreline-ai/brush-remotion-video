# implement_20260717_135202.md

작성 일시: `2026-07-17 13:52:02 KST`

이 문서는 `brush_remotion_video`의 **1차 피아노 BGM 제작 스킬**을 설계·구현하기 위한 개발 계획이다. 지금까지 만든 개별 샘플 피아노 데모의 성과(실제 다중 샘플 렌더·형식/라이선스 provenance)와 실패(화성 불일치·성부 충돌·기계적 어택)를 모두 반영한다. 이 단계의 완료 기준은 “WAV가 생성됨”이 아니라, 원곡·연주·기술·라이선스·사람 청취의 다층 검증을 통과한 **재사용 가능한 피아노 BGM 스킬**이다.

## 개발 목적

- 자연어 요청 또는 선언형 spec으로 길이·용도·장르를 받아 **새로운 솔로 피아노 BGM**을 만든다.
- 완성곡/유튜브 음원/루프를 가져다 붙이지 않고, 로컬의 라이선스 검증된 피아노 다중 샘플과 새 score event만 사용한다.
- 왼손과 오른손을 독립 패턴으로 겹치지 않고, 하나의 화성·성부 진행·프레이즈 계획에서 함께 생성한다.
- 원본 보존용 `48kHz/24-bit WAV`, 사용자 전달용 `44.1kHz/16-bit WAV`, score·provenance·YouTube attribution 후보를 함께 만든다.
- 기존 영상 BGM 정책과 호환되는 manifest를 제공하되, 1차에서는 영상 자동 선택/기존 BGM 교체를 바꾸지 않는다.

## 개발 범위

### 1차 공개 입력 계약

```yaml
projectId: quiet-garden-01
kind: piano-bgm
durationSec: 60                 # 15~600초, 정확한 최종 길이
preset: new-age                 # 아래 preset 중 하나
mood: warm-hopeful
purpose: background             # background | featured
key: G-major                    # 생략하면 preset의 결정적 기본값
tempoBpm: 68                    # 생략 가능
seed: 20260717                  # 생략하면 projectId 기반 결정적 seed
output:
  distribution: youtube         # local | youtube | shorts
  preview: true
```

### 1차 preset

| preset | 음악적 계약 |
|---|---|
| `new-age` | 흐르는 아르페지오·맑은 장/단조·짧은 응답 선율 |
| `ambient-piano` | 긴 여백·낮은 음 밀도·비대칭 잔향 |
| `sleep-piano` | 3/4 또는 6/8 자장가·낮은 다이내믹·해결 중심 |
| `meditation-piano` | 열린 5도·느린 호흡·드론/주파수 톤 없이 피아노만 사용 |
| `lofi-piano` | 7th/9th 보이싱·느슨한 당김음·드럼/노이즈는 제외 |
| `minimal-ambient` | 적은 세포의 점진 변주·명시적 반복 경계 없음 |
| `cinematic-piano` | 넓은 음역·점층·명확한 도착점 |
| `film-ost-piano` | 노래형 선율·도입/전개/종지 구조 |
| `game-bgm-piano` | 짧은 훅·반복 가능한 오스티나토·체크포인트 종지 |
| `fantasy-piano` | 모드 색채(예: Lydian)·고음 장식·안정적 화성 기반 |
| `mystery-horror-piano` | 통제된 반음/트라이톤·침묵·해결/미해결 규칙 명시 |

- preset은 단순 제목/템포 차이가 아니다. 박자·리듬 밀도·화성 어휘·왼손 역할·오른손 역할·형식·다이내믹 곡선이 모두 metadata에 기록되어야 한다.
- 1차는 solo acoustic grand piano만 지원한다. 실제 피아노 샘플 이외의 드럼·패드·빗소리·드론·주파수 톤·외부 loop는 추가하지 않는다.

### 산출물 계약

```text
projects/piano-bgm/<projectId>/
  request.yaml                  # 정규화된 요청
  score.json                    # harmony/phrasing/performance를 가진 원곡 score
  composition-report.json       # 음악 lint 결과·preset/seed/버전

output/original-audio/piano-bgm/<projectId>/
  raw-48k24.wav                 # 실제 sample render
  master-48k24.wav              # 보존/영상 믹스 입력
  <projectId>-44k16.wav         # 사용자 전달 WAV
  preview-30s-44k16.wav         # 30초 초안 청취용, 요청 시
  render-report.json
  provenance.json
  qa.json
  youtube-description.txt
```

- `local-assets/instruments/`에는 실제 Noct-Salamander 원본 WAV/SFZ와 획득 증빙만 둔다. Git/스킬/배포 zip에 원본 샘플을 포함하지 않는다.
- 생성 BGM은 기존 `bgm.mode: asset` 등록을 위한 후보 manifest만 만든다. catalog 등록·자동 BGM 기본값 변경은 사람 청취 승인 뒤 별도 작업으로 한다.

## 확인된 실패와 필수 재발 방지 계약

| 관찰된 문제 | 확인된 원인 | 새 스킬의 강제 계약 |
|---|---|---|
| 반주 위에 고음이 “띵” 하고 튐 | 왼손·오른손을 독립 고정 패턴으로 생성하고, 현재 화음을 검사하지 않음 | 강박 멜로디는 현재 chord의 허용 음만 선택하고, 긴장음은 약박·해결 시점이 score에 명시되어야 함 |
| 판타지 피아노의 불협 | D Lydian인데 F#을 scale 기준음으로 사용; 고음 60개 중 36개가 선언 조성 밖 | `Key`/`Scale` 단일 모델로 tonic pitch class를 계산하고, declared-scale 검사 실패 시 렌더 금지 |
| 보사노바의 이질적 음 | A major인데 C#을 기준음으로 계산; 7개 음급 중 4개가 조성 밖 | key는 문자열이 아닌 typed tonic+mode로 표현하고 A/G/D 등 회귀 fixture를 둠 |
| 일부 새 곡은 기계적·분리된 레이어처럼 들림 | `part`는 metadata일 뿐 renderer가 무시; 랜덤 velocity·고정 release만 적용 | role-aware performance plan, pedal/release, phrase dynamics, accompaniment ducking을 실제 renderer가 소비 |
| 기술 QA PASS인데 음악은 불승인 | 기존 QA가 길이·포맷·peak·샘플 수만 검사 | 음악 lint + render QA + 이어폰/스피커 사람 청취 승인을 모두 완료 조건으로 분리 |
| 수면/명상과 다른 장르가 모두 비슷해짐 | 장르를 제목/tempo만 달리하고 공통 패턴을 재사용 | preset별 meter·harmony·texture·form·note-density 고유성 검사를 둠 |

## 공통 진행 규칙

- 동일 request와 seed는 동일 score·hash를 생성해야 한다.
- source sample은 `assets/instruments/catalog.json`에 등록되고 archive SHA-256·라이선스·출처표시가 provenance에 복사되어야 한다.
- YouTube/Shorts 요청은 기존 BGM 정책의 distribution 규칙을 따르며, CC BY attribution/변경 고지를 delivery text에 포함한다.
- score lint가 FAIL이면 렌더·master·catalog 등록을 진행하지 않는다. 기술 QA만 통과해도 배포 후보가 될 수 없다.
- `mystery-horror`의 불협은 허용 목록·강세 위치·해결/미해결 의도를 score에 선언해야 한다. 다른 preset의 무의도 불협을 정당화하지 않는다.
- prototype scripts/outputs는 회귀 분석 자료로 보존하되 새 CLI의 구현 기반 또는 자동 BGM 자산으로 직접 사용하지 않는다.
- 각 Phase의 자동 테스트 통과 뒤에만 다음 Phase로 진행한다. 사용자 청취 승인 전에는 상태를 `PENDING_USER_LISTENING`으로 기록한다.

## 잔여 리스크 / 후속 과제

- 음악적 자연스러움은 자동 lint만으로 보장되지 않는다. 1차 이후에도 preset별 사람 청취와 timestamp 기반 수정이 필수다.
- Noct-Salamander는 CC BY 3.0 샘플 소스이므로, rendered original composition이라도 attribution/변경 고지 의무를 보존해야 한다.
- Content ID 충돌 가능성을 제거할 수 없다. shared sample source 기반 결과를 Content ID에 등록하지 않는다.
- 다악기/오케스트라/AI 작곡 보조/영상 자동 선택 확대는 1차 승인 후 별도 계획으로 분리한다.

## 구현 검증 결과 (2026-07-17)

- 독립 CLI·schema·11개 preset·화성/성부 lint·실제 Noct-Salamander SFZ renderer·48k/24-bit master·44.1k/16-bit delivery·provenance·generated manifest·사람 청취 gate를 구현했다.
- 11개 preset을 30초씩 실제 샘플로 E2E 렌더해 모두 `musicLint: PASS`, 정확 길이·44.1kHz/16-bit stereo·약 -20 LUFS·최소 8개 이상 샘플 사용을 확인했다.
- 단위/CLI/catalog/installer 대상 검증은 통과했다. 전체 Python suite의 별도 1건 실패는 기존 Melo TTS 환경의 `MeCab` 미설치에 따른 것으로, 피아노 BGM 스킬 변경과 무관하다.
- 대표 파일 `fantasy-garden-30s`는 기술적으로 통과했지만 `PENDING_USER_LISTENING`이다. 사용자가 이어폰·노트북 스피커에서 실제로 듣고 승인하기 전에는 catalog/default BGM으로 등록하지 않는다.
