# implement_20260712_210807.md

작성 일시: `2026-07-12 21:08:07 KST`

이 문서는 `city-watercolor-600s` 실전 수정에서 확인된 완료 연출 펄스와 외부 CC BY BGM 후처리의 수작업을 공용 본선에 환류하되, 기존 brush/pen/pen-brush/shorts/cosmic 프로파일을 바꾸지 않는 범위에서 회귀를 차단하기 위한 작업 문서다.

## 개발 목적

- 일반 `brush-video`의 기본 완료 연출을 `그리기 → 누락 영역 완성 → 밝기 역전 없이 색감 정착 → 정지 감상 → 종이 전환`으로 고정한다.
- 같은 props를 다시 렌더했을 때 소스 기본값 변화로 결과가 달라지지 않도록 완료 관련 값을 명시적으로 직렬화한다.
- 1프레임 번쩍임뿐 아니라 1~2초에 걸친 느린 밝기 역전·확대·블러 중첩도 자동 검수한다.
- 외부 CC BY 음원을 기존 공용 BGM 카탈로그·믹스·라이선스 감사 경로로 처리하고, 완성된 앰비언트 MP4의 BGM 교체도 영상 재렌더 없이 재현 가능하게 한다.
- 최종 산출물에 YouTube 제목·설명·저작자 표시 문구를 함께 생성해 업로드 단계의 누락을 막는다.
- 현재 통과 중인 `Vitest 41건 / pytest 193건 / TypeScript typecheck`와 기존 프로파일 골든을 모두 유지한다.

## 현재 상태 분석

### 유지할 기반

- 얇은 스킬이 공용 `bin/build.py`와 리포 코드를 호출하는 구조는 적절하며 새 렌더러나 독립 BGM 스킬이 필요하지 않다.
- `RevealLayer`에는 `integrated-develop`, 실제 마지막 stroke 종료 기준 timing, `colorSettleFrames`, 마지막 실재 프레임 기준 outro가 이미 존재한다.
- `EffectLayer`는 outro 전에 자연 이펙트를 종료하며, `video-auditor`는 경계·빠른 spike·오디오·라이선스 검사 기반을 이미 갖고 있다.
- 공용 BGM은 catalog/preflight/mix/mux/manifest 구조가 있으므로 새 오디오 엔진이 필요하지 않다.
- 구현 시작 기준은 `typecheck PASS`, `Vitest 41/41`, `pytest 193/193`이었으며, 완료 기준은 `schema sync PASS`, `typecheck PASS`, `Vitest 44/44`, `pytest 209/209`이다.

### 반드시 해결할 불일치

1. `pipeline/brushvid/props.py`의 일반 brush 기본값은 아직 `developFrames: 20`, 전 씬 `prewashOpacity: 0.65`인 legacy 계약이며 `bin/build.py`의 일반 brush 경로가 이를 그대로 사용한다.
2. `src/schema.ts`에는 `colorSettleFrames`, `prewashFadeOutFrames`가 있지만 export된 `schema/render-props.schema.json`에는 누락되어 TS/Python 계약이 갈라져 있다.
3. 현재 city props에는 `colorSettleFrames`가 명시되지 않아 같은 props도 렌더러 기본값 변경 시 결과가 달라질 수 있다.
4. 완료 연출, 색감 정착, 홀드, outro의 시간 예산을 검사하지 않아 짧은 씬에서 서로 겹칠 수 있다.
5. 일반 brush QA는 `start/mid/end` 고정 프레임만 뽑아 실제 완료·정착 구간을 보장하지 않는다.
6. auditor의 일반 spike 검사는 draw 중 큰 seal stroke를 `wash-jump` WARN으로 오인할 수 있고, 느린 밝기 역전은 전용 검사하지 않는다.
7. 자동 선택 BGM은 실제 외부 asset을 사용해도 `cfg.bgm is None`이므로 최종 `--audit`에서 license manifest와 mix report가 전달되지 않는다.
8. 현재 Honor 음원은 로컬 파일과 별도 표시 문구는 있으나 정식 catalog entry가 아니며, 현재 city 매니페스트도 auditor가 요구하는 `assets[]` 표준 구조가 아니다.
9. 최종 제목·YouTube 설명·저작자 표시가 delivery 산출물로 자동 생성되지 않아 업로드 직전에 다시 결정해야 한다.
10. 작업 트리에 관련·무관 변경이 함께 존재하므로 구현 중 `reset`, 대량 포맷, 기존 파일 덮어쓰기를 하면 다른 완료 작업을 손상시킬 위험이 높다.

## 결정 기록

- 새 `completionStyle` 사용자 옵션은 만들지 않는다. 일반 brush의 안전한 기본값을 고치고 특수 효과는 기존 명시 필드로만 opt-in한다.
- 새 렌더러를 만들지 않고 현재 `integrated-develop`를 공용 기본으로 승격한다.
- 일반 brush 기본 prewash는 0으로 한다. 첫 씬 prewash도 사용자가 명시했을 때만 허용한다.
- 완료 관련 렌더 값은 schema default에 기대지 않고 생성 props에 명시한다.
- 완료 시간 예산이 충분하면 `36f develop + 18f settle`을 사용한다. 짧은 씬은 routes를 바꾸지 않고 2:1 비율로 `12f + 6f`까지 결정적으로 축소하며, 최소값으로도 12f hold를 확보할 수 없으면 props 단계에서 명확히 실패시킨다.
- 완료 QA는 props와 routes가 제공될 때만 정밀 실행한다. mp4 단독 auditor의 기존 독립성은 유지한다.
- BGM 교체 CLI는 대사 없는 완성 영상만 지원한다. 내레이션 완성본은 기존 project.yaml의 `--from mix` 경로를 사용한다.
- CC BY 표시는 MP4 metadata만으로 충족했다고 간주하지 않고 YouTube 설명용 원문을 별도 생성한다.
- 기존 골든 기준은 자동 갱신하지 않는다. 변경이 필요한 경우 별도 승인 없이는 실패로 처리한다.

## 목표 기본 프리셋

일반 300프레임 brush 씬의 기본 완료 계약은 다음으로 고정한다.

```json
{
  "completionMode": "integrated-develop",
  "faint": 0.88,
  "developFrames": 36,
  "colorSettleFrames": 18,
  "previewOpacity": 0,
  "prewashOpacity": 0,
  "prewashFrames": 0,
  "prewashHoldFrames": 0,
  "prewashBlur": 0,
  "outroFadeFrames": 24,
  "outroWashOpacity": 1,
  "outroBlur": 0
}
```

시간 계약:

```text
lastStrokeEnd
  → developEnd = lastStrokeEnd + developFrames
  → colorSettleEnd = developEnd + colorSettleFrames
  → minimumHold = 12f
  → outroStart = durationInFrames - outroFadeFrames

colorSettleEnd + minimumHold <= outroStart
```
