# implement_20260713_190836.md

작성 일시: `2026-07-13 19:08:36 KST`

상태: **개발·검증 완료** (`2026-07-13 KST`)

이 문서는 검증이 끝난 Supertonic 여성 음성 10종을 프로젝트의 모든 영상 제작 스킬이 공통으로 사용하는 **버전 고정 음성팩**으로 편입하기 위한 개발 범위를 고정한다. 10개의 개별 스킬을 만들지 않고, 기존 얇은 스킬 구조와 단일 파이프라인 원칙을 유지한다.

## 개발 목적

- 현재 데모로 검증한 여성 음성 10종을 `female-01`~`female-10`이라는 안정적인 프로젝트 음성 ID로 정식 제공한다.
- 기본 음성 F1~F5와 여성 음성끼리만 혼합한 5종을 동일한 `input.tts.voice` 인터페이스에서 선택할 수 있게 한다.
- brush-director와 각 영상 스킬이 콘텐츠 성격에 맞는 음성을 추천하되, 최종 `project.yaml`에는 반드시 명시적인 음성 ID를 기록해 재현성을 보장한다.
- 음성 조합, Supertonic 버전, 모델, 속도, 원본 스타일 해시를 빌드 산출물에 기록해 나중에도 사용 음성을 추적할 수 있게 한다.
- 기존 `F1`~`F5`, `M1`~`M5` 프로젝트와 실더빙 우선 규칙을 깨지 않는다.

## 개발 범위

- `voicePackVersion: 1.0.0` 여성 음성 카탈로그와 JSON Schema 추가
- `female-01`~`female-10` 프리셋 해석기 및 여성 스타일 벡터 혼합 지원
- `input.tts.speed` 파싱·검증과 Supertonic 합성 호출 연결
- 기존 Supertonic 내장 보이스 이름과의 하위 호환
- 로컬 MP3 미리듣기 에셋, 목록/검증/데모 CLI, 브라우저형 청취 페이지 제공
- TTS 스테이지에 해석된 음성 메타데이터와 재현성 manifest 기록
- brush-director 및 TTS를 사용하는 기존 영상 스킬 문서에 음성 선택 규칙 반영
- 단위 테스트, 카탈로그 검증, 대표 프리셋 실합성, 대표 영상 E2E, 기존 F1 회귀 테스트

## 문서 기반 제약 사항

- 스킬은 코드를 내장하지 않는 얇은 안내 계층이며, 실제 구현은 이 저장소의 `pipeline/brushvid`와 `bin`에 둔다.
- TTS 음성의 실제 샘플 길이가 자막과 씬 타이밍의 시계다.
- `input.audio`와 TTS가 함께 있으면 기존처럼 실더빙을 우선하고 TTS는 무시한다.
- Supertonic 미설치·음성팩 오류 시 침묵 폴백을 하지 않고 설치 또는 수정 방법이 포함된 오류를 낸다.
- 공개 영상에는 Supertonic 모델 라이선스 계약에 따라 AI 합성 음성 사용 사실을 고지한다.
- 여성 혼합 프리셋은 F1~F5 스타일 벡터만 사용하며 M1~M5를 혼합 재료로 사용하지 않는다.
- 원본 5종과 혼합 5종을 모두 프로젝트 자체 프리셋으로 버전 고정한다. Supertonic 자체에 F6~F10이 존재한다고 표현하지 않는다.

| 공개 ID | 구성 | 주 용도 |

| `female-04` | F4 100% | 정보 요약·튜토리얼 |
| `female-05` | F5 100% | 힐링·명상·수면·잔잔한 동화 |
| `female-06` | F1 65% + F3 35% | 장편 다큐·차분한 해설 |
| `female-07` | F2 60% + F4 40% | 밝고 명료한 쇼츠 |
| `female-08` | F3 50% + F5 50% | 동화·감성 스토리 |
| `female-09` | F4 65% + F1 35% | 전문 해설·펜 드로잉 설명 |

- `F1`~`F5`는 각각 `female-01`~`female-05`와 같은 결과를 내는 호환 별칭으로 유지한다.
- `M1`~`M5`도 기존 Supertonic 내장 보이스 직접 선택 경로로 계속 허용한다.
- 기존 전역 기본값 `F1`은 회귀 방지를 위해 유지한다. 새로 생성하는 YAML과 문서 예제부터 `female-*` ID를 사용한다.

- QA 스킬과 video-auditor는 음성을 선택하지 않고, 해석된 메타데이터와 고지 여부만 검사한다.

### 3. 런타임 계약

```yaml
input:
  script: narration.txt
  tts:
    engine: supertonic
    voice: female-09
    speed: 1.10
    pauseMs: 350
    timing: tts
```

- `speed` 기본값은 Supertonic 기존 동작과의 호환을 위해 `1.05`, 정식 음성팩 추천값은 `1.10`으로 기록한다.
- 프로젝트 입력은 유한한 양수만 허용하고 운영 안전 범위 `0.70`~`2.00` 밖의 값은 검증 단계에서 거부한다.
- 프리셋 혼합 비율 합은 허용 오차 내 `1.0`이어야 하며 음수·0 비중·알 수 없는 구성 음성을 거부한다.

### 4. 자산·재현성 정책

- 저장소에는 청취용 MP3 10개만 `assets/voices/previews/`에 보관한다.
- 무손실 WAV는 `scripts/generate-supertonic-female-demo.py`로 필요할 때 재생성하며 저장소 자산으로 복제하지 않는다.
- 카탈로그 최상위에 `voicePackVersion`, 엔진, 모델, 언어, 샘플레이트, 기본 속도, 라이선스·AI 고지 정보를 기록한다.
- 빌드 시 `data/{projectId}/tts/voice-manifest.json`과 `data/{projectId}/stages/stt.json`에 다음 정보를 남긴다.
  - `voicePresetId`, `requestedVoice`, `voicePackVersion`
  - `engine`, `packageVersion`, `model`, `language`
  - `components`, `speed`, `pauseMs`
  - 카탈로그 SHA-256 및 사용한 기본 style JSON SHA-256
- 오디오 바이너리의 완전 동일성은 OS·라이브러리 차이로 보장하지 않는다. 조합·버전·해시와 음향 품질 기준으로 재현성을 판정한다.
