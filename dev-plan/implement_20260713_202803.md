### 4. 실전 제작에서 확정된 보존 규칙

- 펜 계열은 원본 잘림을 막기 위해 contain과 종이색 패딩을 사용한다.
- pen-brush는 추출 outline과 원본 선을 중복 합성하지 않으며 최종 선 두께를 원본 수준으로 복원한다.
- 씬 전환은 단순 fade가 아니라 동일한 paper 상태로 수렴해야 하며 중간 씬 prewash를 사용하지 않는다.
- develop 중 레이어 조기 unmount와 이미지 decode 전 캡처를 금지한다.
- QA는 start/mid/end만 보지 않고 lastStrokeEnd, develop, outro 전후를 phase-aware하게 캡처한다.
- BGM은 -23 LUFS 정규화, profile gain, limiter, 음성 활성/비활성 구간 덕킹, license manifest를 함께 본다.
- dark random brush는 mask 0.991과 visible content 0.985를 분리 측정하고 historical cosmic runtime을 유지한다.
- 600초 장시간 영상은 segmented render와 resume 단위가 있어야 한다.
- storybook은 중성 paper 정규화, 씬별 TTS, 고정 씬 길이, pen-brush outro를 함께 적용한다.
- TTS는 명시적 `female-*` ID와 speed, pack/package/model/style hash, AI 고지를 manifest에 남긴다.

## 결정 기록

### 1. 스킬 수는 9개로 유지

- 6개 제작 스킬, 1개 설계 스킬, 1개 파이프라인 QA, 1개 독립 auditor 구조를 유지한다.
- 음성·BGM·전환 검사는 공통 capability/contract이며 신규 스킬로 만들지 않는다.

### 2. 카탈로그가 목록의 단일 진실

`skill/catalog.json`의 각 항목은 최소 다음 필드를 가진다.

```json
{
  "id": "pen-brush-video",
  "folder": "pen-brush-video",
  "role": "production",
  "entrypoint": "bin/build.py",
  "profiles": ["pen-brush"],
  "formats": ["youtube", "shorts"],
  "defaultVoice": "female-09",
  "dependsOn": ["brush-video"],
  "version": "1.0.0",
  "interface": {
    "displayName": "Pen Brush Video",
    "shortDescription": "...",
    "defaultPrompt": "..."
  }
}
```

- 폴더명과 public ID가 다른 `director→brush-director`, `qa-review→brush-qa-review` 매핑을 카탈로그에서 명시한다.
- `cosmic-random-brush-video`는 정식 항목이 아니라 `dark-random-brush-video`의 legacy install alias로만 기록한다.
- README 표와 설치 목록은 카탈로그에서 생성하고 직접 항목을 중복 작성하지 않는다.

### 3. 설치 호환 정책

- `bin/install-skills.sh` 무인자 호출은 기존처럼 Claude만 설치한다.
- `--target claude`, `--target codex`, `--target all`을 명시적으로 지원한다.
- 실제 홈 디렉터리에 쓰기 전 `--dry-run`과 temp HOME 테스트를 필수로 한다.
- legacy cosmic 경로가 symlink일 때만 제거하고 실제 파일·디렉터리면 경고 후 유지한다.
- 끊어진 symlink도 안전하게 감지해 dark 링크로 교체한다.
- 동일 명령을 두 번 실행해도 결과가 같아야 한다.

### 4. 공통 계약 이동 정책

- 공통 계약의 canonical 위치는 `skill/_shared/references/`로 정리한다.
- 첫 정리에서는 기존 `skill/brush-video/references/*` 경로를 즉시 삭제하지 않고 새 canonical 문서로 연결하는 얇은 호환 문서를 남긴다.
- profile 고유 fidelity contract는 각 스킬 폴더에 유지한다.
- 한 릴리스 이상 모든 링크가 새 경로를 사용한 뒤 호환 문서 제거 여부를 별도 계획으로 판단한다.

### 5. 역사 기록 보존 정책

- 기존 dev plan과 FIELD-LOG 항목의 과거 테스트 수치, 당시 이름, 체크박스를 수정하지 않는다.

- 반복 가능한 현재 규칙은 `docs/production-lessons.md`에서 rule ID로 요약하고 FIELD-LOG 원문으로 역링크한다.

## 공통 진행 규칙

- 각 Phase는 앞선 Phase의 자체 테스트 완료 후에만 시작한다.
- 구현 중 발생한 이슈는 해당 Phase에서 수정하고 기록한다.
- 체크박스 상태를 실제 진행 상태와 맞게 업데이트한다.
- 문서에 없는 범위 확장은 하지 않는다.
- 현재 작업 트리는 다수의 미커밋 변경을 포함하므로 `git reset`, `git clean`, 광범위한 `git restore`를 사용하지 않는다.
- 수정 전 각 대상 파일의 hash와 diff를 기록하고 이번 범위의 줄만 최소 수정한다.
- 다른 작업에서 추가·수정한 파일을 덮어쓰지 않는다. 시작 직전 `git status`와 파일 mtime을 다시 확인한다.
- 설치 테스트는 먼저 temp HOME에서 수행하고 실제 `~/.claude`, `~/.codex` 반영은 명시적 승인 뒤에 한다.
- 카탈로그·문서·설치 정리 때문에 영상 런타임 코드를 변경하지 않는다.
- 자동 생성 파일은 `--check` 모드를 제공해 CI에서 드리프트를 검출한다.
- 생성 결과는 결정적이어야 하며 동일 입력에서 byte-identical해야 한다.

- 최종 완료 전 기준선 전체를 다시 실행하고 하나라도 실패하면 완료 처리하지 않는다.
