## 개발 목적

- 정지 이미지의 원래 색을 유지하며 2D 모션·효과·BGM을 적용하는 Full Color Motion V1을 기존 파이프라인에 통합한다.
- V2에서 내레이션/TTS cue, 타이틀/자막, Shorts 세로 포맷을 같은 프로파일에 연결한다.
- 마지막 선택 옵션으로 정적 이미지가 붓 자국으로 나타난 뒤 Full Color Motion으로 이어지는 brush reveal을 제공한다.
- `full-color-motion-video`를 코드 사본 없는 독립 생산 스킬로 등록한다.

## 개발 범위

- `drawing.profile: full-color-motion`과 strict `motion:` YAML 계약
- Motion 전용 Zod/JSON Schema와 props 빌더, 가로/세로 Remotion composition
- 2D movement 8종, effect 12종, 12f crossfade, 이미지별 강도·리빌 옵션
- V1: user-images + ambient/BGM + 가로 FHD/UHD
- V2: narration/whisper/TTS cue, title/subtitle, Shorts FHD 세로와 safe zone
- `reveal: brush`일 때 기존 routes 생성 결과를 재사용하는 Full Color Motion brush-mask overlay와 cursor
- stage cache·render·mix·mux·QA·audit의 profile 분기와 예제/스킬/문서

## 결정 기록

- YAML은 `motion.movement`를 사용한다. 이 값은 정지 이미지의 2D Ken-Burns 프리셋이며 AI 카메라 프롬프트나 true 3D camera가 아니다.
- V1과 V2의 props는 brush props와 분리된 `FullColorMotionPropsSchema`를 사용한다. 브러시 전용 필드를 Motion schema에 섞지 않는다.
- V1은 가로와 BGM만, V2에서 음성·자막·세로를 확장한다. 실제 빌드는 공통 props/renderer로 수행한다.
- brush reveal은 `motion.reveal: brush` 명시 시에만 활성화하며, 기존 `routes` 생성기를 사용한다. 원본 이미지 자체를 흰 종이 clean 처리하지 않는다.

## 공통 진행 규칙

- 각 Phase는 앞선 Phase의 자체 테스트 완료 후에만 시작한다.
- 구현 중 발견한 이슈는 해당 Phase에서 수정하고 체크박스와 이슈란을 갱신한다.
- 현재 작업트리의 piano/seamless 및 사용자 변경을 보존하며 reset/restore/대량 포맷을 사용하지 않는다.
- schema는 TypeScript Zod가 유일한 정의이며 JSON Schema는 export만 한다.
- 새 스킬에는 렌더 코드 사본을 넣지 않고 공용 `bin/build.py`만 호출한다.
- 로컬 검증은 typecheck/schema/test와 작은 실제 렌더+audit으로 수행한다.
