# implement_20260712_124551.md

작성 일시: `2026-07-12 12:45:51 KST`

이 문서는 현재 사용자 승인을 받은 우주 일출 랜덤 브러시 1씬을 골든 기준으로 고정하고,
이를 코드 사본 없는 독립 프로파일 스킬 `cosmic-random-brush-video` v0.1로 본선에 편입하기 위한 개발 범위를 정한다.

## 개발 목적

- 현재 데모의 **자유 랜덤 터치 브러싱**을 재현 가능한 본선 프로파일로 옮긴다.
- 사용자 접점은 독립 스킬 `cosmic-random-brush-video`로 제공한다.
- 실행은 기존 `bin/build.py`, 공용 Python 파이프라인, Remotion 엔진을 재사용한다.
- 스킬 폴더에는 Python·TypeScript 렌더러 사본을 넣지 않는 **얇은 스킬** 원칙을 지킨다.
- `project.yaml`의 프로파일 선택만으로 이미지 1장 → 10초 MP4 → QA 산출물까지 생성한다.
- 커버리지가 부족할 때 붓을 키우지 않고, **동일 크기의 보완 터치**를 추가해 99.1% 이상을 달성한다.
- 현재 승인 데모와 시각·수치가 달라지는 브러싱 방식 회귀를 자동으로 차단한다.

## 개발 범위

- **독립 프로파일 계약**
  - 스킬명: `cosmic-random-brush-video`.
  - `drawing.profile: cosmic-random-brush`를 추가한다.
  - v0.1은 `format: youtube`, 1920×1080, 30fps, 씬당 300프레임만 지원한다.
  - 입력은 준비된 이미지 1장 또는 기존 `background.strategy`가 만든 이미지 1장이다.
- **랜덤 터치 routes 본선화**
  - 기본 자유 터치 36개를 결정적 seed로 생성한다.
  - 붓 폭은 1920×1080 기준 230~365px 범위를 벗어나지 않는다.
  - 남은 영역의 최대 미커버 지점을 분석해 같은 폭 범위의 보완 터치를 추가한다.
  - 전체 경로 마스크 커버리지 0.991 이상에서 종료한다.
  - 보완 터치는 최대 20개로 제한하며, 목표 미달 시 조용히 확대하지 않고 빌드를 실패시킨다.
- **다크 랜덤 브러시 렌더러 본선화**
  - 현재 데모의 거친 브리슬 마스크, 짧은 터치, 큰 위치 이동, 이동 중 보이는 붓을 공용 Scene 레이어로 옮긴다.
  - 기존 `brush`, `pen`, `pen-brush` Reveal/Cursor 경로는 변경하지 않는다.
  - routes의 `meta.family`로 기존 리빌과 랜덤 터치 리빌을 명확히 분기한다.
  - 다크 네이비 배경, 별, 미세 발광, 최종 홀드, 다크 아웃트로를 프로파일 전용 레이어에서 처리한다.
- **파이프라인 통합**
  - background/clean/routes/props/render/mix/mux/qa 스테이지가 신규 프로파일을 처리한다.
  - 기존 스테이지 캐시와 `--from routes|props|render|mix|qa` 재개 계약을 유지한다.
  - 기존 공용 BGM·믹스·라이선스 정책을 그대로 사용한다.
- **골든 회귀 fixture**
  - 승인된 우주 일출 원본, seed, route 메트릭, 핵심 프레임을 v0.1 골든 기준으로 보존한다.
  - 동일 입력·seed를 두 번 실행했을 때 routes JSON과 핵심 프레임이 결정적으로 동일해야 한다.
  - 골든 업데이트는 명시적 명령으로만 허용한다.
- **얇은 스킬·설치·문서 편입**
  - `skill/cosmic-random-brush-video/SKILL.md`와 필요한 reference만 추가한다.
  - `bin/install-skills.sh`에 symlink 설치 항목을 추가한다.
  - README 스킬 카탈로그와 director intent map에 신규 프로파일을 등록한다.

## 문서 기반 제약 사항

- 이 리포가 실행 코드의 유일한 진실이다. 스킬에는 엔진 코드를 복사하지 않는다.
- `src/schema.ts`가 render-props 데이터 계약의 유일한 정의다.
- Python 파이프라인은 export된 JSON Schema를 소비하며 별도 스키마를 이중 정의하지 않는다.
- 신규 프로파일은 기존 단일 routes와 `drawingPhases` 계약을 강제 마이그레이션하지 않는다.
- 기존 공용 BGM·mux·QA·video-auditor 경로를 우선 재사용한다.
- 새 의존성을 추가하지 않는다. 현재 Pillow, numpy, scipy, React, Remotion 범위에서 구현한다.
- 스킬 설치는 사본이 아닌 symlink로 수행한다.

## 결정 기록

- **스킬을 먼저 추가하되 v0.1은 승인된 1씬의 본선화까지만 수행한다.**
- 스킬은 독립 사용자 접점을 제공하지만 실행 엔진은 공유한다.
- 프로파일 값은 `cosmic-random-brush`, 스킬 폴더명은 `cosmic-random-brush-video`로 고정한다.
- v0.1의 입력은 1씬·이미지 1장으로 제한한다.
- routes 계약은 기존 `RoutesDataSchema`를 느슨하게 확장하지 않고, 랜덤 터치 전용 schema를 별도로 둔다.
- raw JSON 로드 후 `meta.family`를 확인하고 해당 schema로 검증한 뒤 렌더러를 분기한다.
- 기존 브러시 리빌과 랜덤 터치 리빌은 같은 React 레이어에 조건문을 누적하지 않고 책임 단위로 분리한다.
- 커버리지 해결 순서는 `기본 36터치 → 미커버 분석 → 동일 폭 보완 터치 → 0.991 게이트`로 고정한다.
- 보완 횟수가 20개를 넘기거나 0.991에 도달하지 못하면 이미지/경로 문제로 실패한다.
- 최종 bitmap 페이드는 붓질 완료를 대신하는 커버리지 수단으로 사용하지 않는다.
- 현재 데모 파일은 본선 parity가 확인될 때까지 유지하고, 마지막 Phase에서만 중복 제거 여부를 판단한다.

## 공통 진행 규칙

- 각 Phase는 앞선 Phase의 자체 테스트 완료 후에만 시작한다.
- 구현 중 발생한 이슈는 해당 Phase에서 수정하고 기록한다.
- 체크박스 상태를 실제 진행 상태와 맞게 업데이트한다.
- 문서에 없는 범위 확장은 하지 않는다.
- 요구사항이 불명확하면 가능한 케이스를 나누고 확인 후 진행한다.
- 한 기능은 유지보수 가능한 최소 책임 단위로 나누고 역할과 경계를 지킨다.
- 계획 밖 옵션·확장·부가 기능을 위한 추상화는 만들지 않는다.
- 공통화는 데모 중복을 본선으로 옮길 때만 수행하고 기존 프로파일을 함께 재작성하지 않는다.
- 기존 프로젝트 API, 공용 빌드 단계, 현재 의존성을 우선 사용한다.
- render-props 데이터 계약 변경은 `src/schema.ts`에서만 시작한다.
- 신규 routes는 로드 즉시 schema 검증하며 잘못된 family/point/타이밍을 침묵 통과시키지 않는다.
- 골든 기준 변경은 사용자 승인이나 별도 계획 없이 수행하지 않는다.
- 최종 MP4는 QA와 video-auditor가 모두 PASS한 경우에만 완료로 처리한다.
