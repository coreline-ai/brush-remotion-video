## 개발 목적

기존 `bin/piano-bgm.py`에 Stable Audio 3 MLX를 명시적·자동 선택 가능한 생성 엔진으로 통합하고,
기존 Noct-Salamander 샘플 score 경로를 호환 가능한 fallback으로 보존한다.

## 개발 범위

- `piano-bgm` request schema의 `engine`, prompt, sampling 계약
- `stable-audio-3-mlx` 로컬 adapter와 환경변수 기반 실행 경로

- 공통 후보 WAV, provenance, 기술 QA, 사람 청취 승인 gate
- 기존 `sample-score` output/CLI의 backward compatibility
- 관련 단위·통합 테스트, 스킬·README·catalog 문서 갱신

## 결정 기록

- 엔진 값은 `auto | stable-audio-3-mlx | sample-score`로 고정한다.
- 기존 YAML에서 `engine`을 생략하면 `sample-score`로 해석해 기존 결과의 재현성을 보존한다.
- 새 요청에서 `engine: auto`를 사용하면 `featured` 또는 cinematic/film/game/fantasy/mystery preset은 Stable Audio를 우선하고, sleep/meditation/minimal은 sample-score를 선택한다.
- 명시적 Stable Audio 요청에서 런타임/가중치가 없으면 silent fallback하지 않고 명확히 실패한다. `auto`만 설치 누락 시 sample-score로 fallback하고 사유를 manifest에 기록한다.
- Stable Audio는 symbolic score를 만들지 않으므로 `compose`는 sample-score 전용으로 유지하고, 생성 분기는 `generate`/`build`에서 처리한다.
- 모델 경로는 `/Volumes/...`를 코드에 하드코딩하지 않고 `STABLE_AUDIO_3_MLX_ROOT` 환경변수로 주입한다.
- `generated-bgm-manifest.json`이 `APPROVED`가 되기 전에는 기존 catalog·자동 선택·YouTube delivery에 연결하지 않는다.
