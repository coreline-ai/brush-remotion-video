# implement_20260717_120502.md

작성 일시: `2026-07-17 12:05:02 KST`

이 문서는 무료 CC BY 피아노 멀티샘플을 기반으로, 전자음이 아닌 샘플 기반의 독창적 수면 피아노 곡을 만들고 기존 BGM 배포·검수 흐름에 안전하게 연결하기 위한 작업 계획이다.

## 개발 목적

- 기존 사인파 기반 절차적 피아노 합성을 대체할 수 있는 샘플 기반 렌더 경로를 만든다.
- 완성곡을 재사용하지 않고, 새 MIDI 작곡·연주 데이터로 1분 이상의 독창적 피아노 트랙을 만든다.
- 무료 샘플의 원본·라이선스·해시·변경 사실을 기록해 YouTube 배포 시 저작자 표시와 검증 근거를 남긴다.
- 최종 데모는 스테레오 PCM WAV `44.1kHz / 16-bit`로 출력하고, 청취·기술 검수 후에만 기존 BGM 시스템에 등록한다.

## 개발 범위

- CC BY 라이선스의 Noct-Salamander/Accurate-Salamander 피아노 멀티샘플의 출처·라이선스·다운로드 해시를 로컬 manifest로 고정
- 원본 샘플 파일은 `local-assets/instruments/`에만 보관하고 Git에는 manifest·코드·문서만 보관
- 샘플의 음높이·velocity·릴리스 정보를 읽어 MIDI note event를 렌더하는 최소 책임 단위의 renderer 구현
- 독창적 1분 수면 피아노 MIDI/연주 데이터와 `44.1kHz / 16-bit stereo PCM WAV` 출력
- 생성 BGM의 provenance(샘플 출처, 라이선스, 렌더러 버전, MIDI/오디오 hash, 기술 메타데이터) 생성
- `assets/bgm` 카탈로그와 preflight/audit/delivery가 생성곡의 CC BY 저작자 표시를 누락 없이 처리하도록 최소 확장
- 자동·단위·통합·청취 QA 및 1분 데모 결과물

## 제외 범위

- 타인의 완성곡·유튜브 영상·데모곡에서의 음원 추출 또는 루프 재사용
- 피아노 녹음 세션을 새로 진행해 88건반을 직접 샘플링하는 작업
- 외부 유료 VST·DAW·클라우드 작곡 API의 도입
- 10분/수시간 버전, 다악기 편곡, 빗소리·드론·주파수 톤 추가
- Remotion 장면·드로잉·TTS·기존 BGM 전체의 무관한 리팩터링
- 샘플 원본 WAV/SFZ나 대용량 다운로드 파일을 공개 Git tree에 포함하는 일

## 문서 기반 제약 사항

- 빌드·렌더 시점에는 인터넷 다운로드를 하지 않는다. 샘플은 명시적 획득 단계에서만 내려받고, 이후 렌더는 로컬 파일만 사용한다.
- Pixabay 자산은 YouTube/Shorts 신규 배포에 쓸 수 없다. 이번 자산은 Pixabay와 무관한 CC BY 샘플 기반의 새 작곡물로 관리한다.
- CC BY의 저작자 표시와 변경 사실은 생성곡 metadata, BGM license manifest, delivery의 YouTube 설명문에 남긴다.
- 원본 샘플을 재배포하거나 샘플팩으로 내보내지 않는다. 최종 곡 WAV만 BGM asset으로 등록한다.
- 전체 음원은 현재 BGM 파이프라인의 48kHz 작업 WAV와 별개로, 사용자 전달용 master를 `44.1kHz / 16-bit` PCM WAV로 보존한다. 영상 믹스 단계의 48kHz 변환은 기존 계약을 유지한다.
