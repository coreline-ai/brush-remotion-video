# 스토리·SRT·TTS 계약

## 1. 시간 모델

기본값은 `10씬 × 10초 = 100초`다. 다른 길이도 다음 식으로 명시한다.

```text
total_seconds = scene_count × scene_seconds
total_frames = total_seconds × 30
```

쇼츠는 180초를 넘기지 않는다.

## 2. 장면 이야기 구조

10씬 기본 역할:

1. 세계와 사건
2. 발견
3. 선택
4. 첫 실패
5. 다시 시도
6. 위기
7. 보상 시작
8. 마법의 절정
9. 친절의 확산
10. 결말과 교훈

장면마다 하나의 행동과 하나의 감정을 사용한다.

## 3. 파일 계약

### narration.txt

- 비어 있지 않은 한 줄을 한 씬으로 해석한다.
- 기본은 한 줄에 짧은 두 문장이다.
- 문장 1은 상황·행동, 문장 2는 감정·결과를 담당한다.

### subtitles.srt

- 모바일 가독성을 위해 한 씬에 짧은 cue 2개를 권장한다.
- 한 cue는 최대 2줄이다.
- 장면당 기준 배치는 다음과 같다.

```text
cue 1: scene_start + 0.4s
cue 2: 첫 문장 종료 + pause
scene tail: 최소 0.6s
```

수동 SRT는 검토 기준이다. 최종 렌더에는 합성된 실제 음성 길이로 다시 계산한 SRT를 사용한다.

## 4. Supertonic

- 새 프로젝트 기본 음성: `female-08` (F3 50% + F5 50%, 동화 추천)
- 기존 검증 예제의 `F1`은 하위 호환 별칭으로 유지한다.
- 공통 10종 선택표: [Supertonic 음성 카탈로그](../../_shared/references/supertonic-voice-catalog.md)
- 기본 언어: 한국어
- 기본 합성 속도: `1.10`
- 문장 사이 쉼: `350ms`
- 톤: 젊고 밝은 여성, 아이에게 책을 읽는 속도
- 감정: 초반 호기심, 중반 긴장, 후반 미소와 안정

## 5. 씬별 고정 블록 합성

전체 대본을 한 번에 100초로 늘이지 않는다. 각 씬을 독립 합성한다.

```text
scene_seconds = 10.0
lead = 0.4
tail = 0.6
max_speech = 9.0
```

- 음성이 9초 이하이면 원래 속도를 유지하고 남는 구간을 무음으로 채운다.
- 9초를 넘으면 필요한 `atempo`를 계산한다.
- `atempo > 1.15`이면 보정하지 말고 대본을 줄인다.
- 각 장면 블록을 정확히 10초로 패딩한 뒤 순서대로 결합한다.
- 결과 음성은 정확히 100초, 장면은 각 300프레임이어야 한다.

## 6. 씬 매핑

기본 `group_scenes()`가 20개 문장 cue를 임의 장면으로 묶게 두지 않는다.

- 이미지: `1 scene = 1 image`
- 음성: `1 narration line = 1 scene`
- 자막: `2 sentence cues = 1 scene`
- 씬 길이: 고정 300프레임

`scripts/prepare-storybook-full-touch.py`가 `scenes.json`과 `stt/cues/background` 스테이지 마커를 생성해 이 매핑을 고정한다.

## 7. 읽기 품질 체크

- [ ] 고유명사 발음이 일관됨
- [ ] 씬 시작 전에 말하지 않음
- [ ] 다음 씬으로 문장이 넘어가지 않음
- [ ] 과도한 속도 보정 없음
- [ ] 자막이 음성과 같은 문장 단위
- [ ] 마지막 문장 뒤 최소 0.6초 여운
