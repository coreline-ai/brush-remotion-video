Overview

30초 큰 그림 — 무엇이 들어가서, 무엇이 나오나

세부를 보기 전에 전체 형태부터. 시스템은 “재료 → 파이썬 파이프라인 → Remotion 렌더”의 한 방향 흐름입니다. 이하 해부 섹션(원리~입력 모드)은 참조 시스템 new-video-gen 기준이고, 새 프로젝트의 설계는 마지막 두 섹션(재설계 방향·로드맵)에 있습니다.

  • 주제 텍스트 (배경 그림 생성용)
  • 또는 아무것도 없이 — 앰비언트 모드

Reveal Engine

그려지는 원리 — 붓선은 “그림을 드러내는 마스크”다

핵심 아이디어 하나만 이해하면 됩니다. 배경 그림(잉크+수채화 PNG)은 처음엔 숨겨져 있고, 붓이 지나간 자리만 SVG dash-offset 마스크로 연하게(faint 0.6~0.72) 드러납니다. 그림이 다 그려지면 전체가 또렷하게 “발현(develop)”됩니다.

한 씬(기본 300프레임 = 10초)의 4박자 타임라인

① prewash — 은은한 예고
② 붓 리빌 — drawStart→drawEnd, faint
③ develop + 홀드
④ outro
0f8f220f228f + developFrames300f

씬을 조율하는 손잡이들 (per-scene props)

BrushDrawScene(760줄)이 읽는 주요 파라미터 — 전부 JSON으로 제어합니다.

faintedgeFeatherlinearDraw developFramesprewash*(4)outro*(3) brushDynamics{speed·jitter·seed}naturalEffects{6종}

naturalEffects: mist · streamSparkle · forestDust · meadowWind · starTwinkle · sunsetGlow

화면 위 나머지 요소

  • PenCursor — 스트로크 선두를 따라다니는 붓 이미지 (brush.png + tip 좌표)
  • TopTitle — 골드 kicker + 제목, 첫 단어 색은 배경에서 자동 추출
  • Subtitles — 하단 자막 band, cue{text, from, to} 프레임 단위
  • 위젯 3레이어 — 빈 여백에 배치되는 카드·차트 (아래 섹션)

클린 처리된 잉크+수채 그림. render-props에는 직접 등장하지 않고 routes를 통해서만 도달

자막은 SRT 파일 참조가 아니라 props 안의 frame 기반 cues[]로 들어갑니다 — SRT는 빌드 단계의 재료. render-props-video-only.json은 같은 파일에 audio만 null (영상 먼저 렌더 후 오디오 mux용).

스키마는 5세대를 거쳤습니다. 단일 씬(10초) → 멀티 씬 → 타이틀·위젯 → prewash·naturalEffects → 장편·transition·brushDynamics·whiteWidgets(현재). 구세대 필드는 컴포넌트가 optional로 처리하므로 옛 프로젝트 props도 그대로 렌더됩니다 (하위호환). 새 프로젝트는 이 하위호환을 승계하지 않습니다 — Zod 스키마 v1(schemaVersion: 1)로 초기화하고, 옛 props 재렌더는 new-video-gen에 남깁니다. 단, routes JSON 포맷은 호환 유지(골든 샘플 재사용).
Scripts

파이썬 스크립트 지형도 — 재사용 코어는 작고, 복붙은 크다

외부 의존성 전부 로컬·무료: PIL · numpy · scipy · scikit-image · ffmpeg/ffprobe · 로컬 whisper(한국어 STT) · codex 내장 image_gen(chatgpt 인증) · Remotion(Chromium). API 키가 필요한 단계가 하나도 없습니다.
Input Modes

입력 모드 2가지 — 자막이 있느냐 없느냐

같은 파이프라인이지만 “씬 타이밍의 출처”가 다릅니다. 새 프로젝트가 흡수해야 할 두 경로입니다.

A. 내레이션 모드 — SRT가 시계다
  1. 더빙.mp3 → 로컬 whisper(small·ko) → 초벌 SRT
  2. SRT 타임스탬프 교정 → 자막 테이블 확정
  3. 씬 경계 = SRT 구간 → durationInFrames = (end−start)×30
  4. 자막 문장 = cues (길면 split-cues로 분할)

예: adaptive-html-brush-white (18씬·124초), 설명형·교육형 영상

B. 앰비언트 모드 — 시계는 고정 10초
  1. SRT 없음 — 씬 = 300프레임(10초) 고정 × N
  2. 배경: PIL 절차 합성 or image_gen 시트 분할
  3. BGM: numpy/wave로 피아노·자연음 직접 합성 (저작권 프리)
  4. cue는 시적인 한 줄 문구를 수동 배치

예: winter-snow-pine(60씬·600초), relaxing-nature 시리즈(최대 1000초), 힐링·롱폼

Rebuild Strategy

재설계 방향 — 이전(verbatim)이 아니라, 완전 신규 제작

2026-07-11 확정. 위 해부가 보여준 부채(씬 모놀리스 761줄 · 3중 위젯 체계 · 5세대 스키마 · 복붙 1.4만 줄)를 새 리포로 옮기지 않습니다. 코드는 전부 새로 쓰고, new-video-gen은 읽기 전용 스펙 소스로만 씁니다. 초기의 verbatim 이전 계획(MIGRATION-PLAN.md)은 분석 자료로만 남습니다.

전략을 바꾼 결정적 근거 — 사본 드리프트 실측 (2026-07-11): 스킬에 내장된 코드 사본은 390줄로 라이브(761줄)보다 한 세대 뒤졌고, 위젯 68종 레이어는 아예 누락, 파이썬 헬퍼도 3종(place-widgets · validate-layout · pick-title-color)이 어긋나 있었습니다. “복사해서 유지”는 이미 실패가 증명된 방식 — 그래서 새 리포가 유일한 소스, 스킬은 코드 사본 0개가 제1원칙입니다.
새로 설계 (코드)

레이어별 소형 컴포넌트 — Reveal·Cursor·Subtitle·Title·Effect로 761줄 모놀리스 해체. 위젯은 단일 registry 핵심 10~15종(실사용 빈도 기반), Zod 스키마 v1이 유일한 진실, 파이썬은 brushvid 패키지 + bin/build.py 단일 진입점.

스펙으로 채택 (값·알고리즘)

수많은 반복으로 튜닝된 이징 곡선·기본값(faint 0.6 등)·brushDynamics 수식은 라이브 코드에서 읽어 채택. routes JSON 포맷은 호환 유지 — 검증된 골든 샘플(winter-snow-pine)을 테스트 입력으로 재사용. skeletonize→RDP→seal 알고리즘도 동일 개념으로 재구성합니다.

버리는 것 (명시적으로)

구세대 하위호환 필드(meta{} · 레거시 widgets[]) · 3중 위젯 분리 체계 · 빌더 42종 복붙(아카이브조차 안 가져옴) · 스킬 내 구버전 코드 사본(참고도 금지). 옛 프로젝트 재렌더는 new-video-gen에 남습니다.

brush_remotion_video/ ├─ src/schema.ts ★ Zod 스키마 v1 = 유일한 진실 (schemaVersion · JSON Schema 내보내기) ├─ src/Root.tsx 컴포지션 2종 — BrushLandscape(16:9) · BrushPortrait(9:16) ├─ src/scene/ BrushScene(조립 전용) + Reveal · Cursor · Subtitle · Title · Effect · Sequence ├─ src/lib/ geometry · dynamics · easing (순수 함수 — 단위 테스트 대상) ├─ src/widgets/registry.ts 단일 위젯 체계 (Zod discriminated union → 컴포넌트 매핑) ├─ pipeline/brushvid/ routes · background · cues · stt · layout · audio · props · render · qa ├─ bin/build.py ★ 단일 진입점 — project.yaml → mp4 + QA ├─ tests/golden/ 기준 스틸 + diff.py (픽셀 오차 임계 2%) └─ skill/ 초기 설계 당시 얇은 스킬 2종 — 현재는 catalog 기반 9종, 코드 사본 0
# project.yaml — 최종 사용자 경험: 이 파일 하나로 영상 완성
projectId: my-video
format: youtube            # youtube(16:9) | shorts(9:16)
input:
  audio: 더빙.mp3          # → whisper → SRT → 씬/자막 자동 · 둘 다 없으면 앰비언트 모드
background:
  strategy: imagegen       # imagegen | preset(PIL 폴백) | user-images
widgets: auto              # auto = 빈영역 탐지 + 자동 배치 + 겹침 검증
