  • 더빙.mp3만 (→ whisper가 SRT 생성)

  • Remotion 렌더 → 오디오 mux → QA 캡처

출력
  • output/{projectId}.mp4 (16:9 / 9:16)

설계 제1원칙: 이 리포가 유일한 소스입니다. new-video-gen은 읽기 전용 스펙 참고였고 코드는 전부 신규 작성 — 튜닝된 값(이징 곡선·faint 0.6 등)과 routes JSON 포맷만 채택했습니다. 스킬도 코드 사본 없이 이 리포를 실행 대상으로 지정합니다 (드리프트 원천 차단).
Build Pipeline

빌드 10 스테이지 — 각 단계는 캐시되고, 어디서든 재개된다

전부 pipeline/brushvid/ 파이썬 모듈. 스테이지 산출물은 data/{pid}/stages/에 기록(StageLedger)되어 --from <stage>로 실패 지점부터 재개합니다.

1·2

stt → cues

brushvid.stt · brushvid.cues

audio만 있으면 로컬 whisper(small·ko)로 SRT 생성. SRT를 fps30 frame으로 환산하고, 긴 문장은 한글 글리프 폭 비례로 분할, 씬 경계로 그룹핑(길이 상·하한).

더빙.mp3 / SRT→씬 타이밍 + cues[]
3·4

background → clean

brushvid.backgroundAPI 키 불필요

전략 3종 — imagegen(codex exec 내장), preset(PIL 절차 합성·시드 결정적, imagegen 불가 시 자동 폴백), user-images(contain-fit). 이어서 종이색 키잉으로 빈 영역이 리빌되지 않게 전처리.

주제/이미지→씬별 content.png
5

routes — 붓 경로 추출

brushvid.routes파이프라인의 심장

콘텐츠 마스크 → 골격화(skeletonize) → 폴리라인 추적 → RDP 단순화 → seal 밴드로 커버리지 보장 → 이동 최소화 순서 → drawStart→drawEnd 타이밍 배분. 검증 실측: 커버리지 100%, 스트로크 256개(기준 303±20% 내).

content.png→routes.json {meta, strokes[]}
6·7

layout → props

brushvid.layout · brushvid.props검증 FAIL 시 중단

위젯 배치 검증(위젯↔위젯/자막존/타이틀존 겹침 hard-fail, 여백≥90px) 후 render-props v1을 조립하고 schema/render-props.schema.json으로 검증. 이 JSON Schema는 TS Zod에서 내보낸 것 — 파이썬은 소비만 합니다(이중 정의 금지).

routes + cues + widgets→data/{pid}/props.json
8·9

render → mux

brushvid.render

npx remotion render (장편은 세그먼트 렌더+concat). props는 audio null로 렌더한 뒤 ffmpeg로 오디오 합성 — 씬 합산과 오디오 길이가 1초 이상 어긋나면 경고 후 자동 보정.

props.json→output/{pid}.mp4
10

qa — 리뷰 루프의 시작

brushvid.qa · bin/qa.py

씬별 start/mid/end 스틸 캡처 → capture-manifest.json + 콘택트시트. 이 산출물이 brush-qa-review 스킬의 입력 계약이 됩니다 (씬별 이슈 → scene-fix-request JSON → 해당 스테이지 재빌드).

mp4→data/{pid}/qa/contact-sheet.png→수정 → --from 재빌드

입력 모드 3가지 — 전부 E2E 실측 완료

A. 내레이션 — SRT가 시계
  1. SRT 구간 → 씬 경계 + cue
  2. 배경 N장 생성 → routes N개
  3. 제공 오디오 mux

실측: SRT 94항목 → 31씬 · 427초 mp4

B. whisper — 음성만으로
  1. 더빙.mp3 → whisper(small·ko) → SRT
  2. 이후 내레이션과 동일

실측: 합성 음성 정확 전사 → 10초 mp4

C. 앰비언트 — 무입력
  1. 300프레임(10초) × N씬 고정
  2. BGM 합성 (numpy · 시드 결정적)
  3. 시적 한 줄 cue 수동 배치

실측: 3씬 · 30초 mp4 + 합성 BGM

Render Tree

렌더 구조 — 조립은 씬이, 그리기는 레이어가

참조 시스템의 761줄 모놀리스를 단일 책임 레이어들로 해체했습니다. BrushScene은 데이터 로드와 타임라인 파생만 하고(조립 전용), 그리기는 각 레이어가 담당합니다.

src/index.ts ← registerRoot (입구) └─ Root.tsx ← 컴포지션 2종 등록 + calculateMetadata(스키마 parse → 기본값 채움 → duration 합산) └─ SceneSequence ← scenes[] → Sequence 연결 + 공용 Audio 1트랙 (씬 경계 무끊김) └─ BrushScene ← routes 로드(Zod 검증) + drawFrame 파생 + 레이어 조립 ├─ RevealLayer ← faint 리빌 → develop (+prewash/outro/edgeFeather/parallax) ├─ EffectLayer ← naturalEffects 6종 (seed deterministic) ├─ WidgetLayer ← CardShell + registry 바디 15종 ├─ TitleLayer ← 골드 kicker + firstWordColor ├─ CursorLayer ← getPenPose → 붓 이미지 커서 └─ SubtitleLayer ← cues[] + 단어 하이라이트

Widgets

위젯 15종 — 3중 체계를 단일 registry로

참조 시스템의 3중 위젯 체계(레거시 4 + Azure 9 + 카탈로그 50, 레이어 3개)를 CardShell 카드 시스템 + 단일 registry로 통합했습니다. 핵심셋은 추측이 아니라 실사용 12개 프로젝트의 사용 빈도 집계로 확정했습니다.
