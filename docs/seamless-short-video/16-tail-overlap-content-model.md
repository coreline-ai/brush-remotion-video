**관련:** [13](13-common-remediation-standard.md) · [14](14-multi-signal-handoff-architecture.md) · [15](15-ten-second-boundary-common-playbook.md)

## 0. 공통 적용 정본 (이 문서의 한 줄)

```text
이전 씬 마지막 약 2초의 동작을
다음 씬이 그대로 이어 받으면
→ 경계에 불필요한 동작 연출을 넣을 필요가 없고
→ 연결이 자연스럽다.
```

이것이 **씬 경계 연속의 1순위 목표**다.  
다른 처방(continue 문장, auto-trim, tone-match …)은  
**이 목표가 도구로 안 될 때의 근사**이지, 정본을 대체하지 않는다.

### 0.1 왜 이렇게 하면 되는가

| 끊기는 이유 | 2초 동작이 진짜 이어지면 |
| --- | --- |
| 다음 씬이 걸음/속도를 **새로 샘플** | 위상·속도가 이미 이어짐 → 리셋 불필요 |
| 컷 직후 **올려다보기·정지·엔딩 포즈** 등 임의 연출 | 그런 연출을 **넣을 이유가 없음** |
| “연결되게 보이게” 편집 트릭 | **생성 단계에서 이미 연속** |

즉:  
**불필요 동작을 “잘 설계”하는 문제가 아니라,  
이전 2초 동작을 끊지 않으면 애초에 안 넣어도 된다.**

### 0.2 정본이 요구하는 것 (해야 할 일)

```text
입력(이상):  이전 클립의 마지막 ~2초 영상 (모션 tail)
출력:        그 동작이 끊기지 않고 이어지는 다음 10초
경계:        새 스토리 비트·포즈 변경·가속 리셋 금지 (이어지기만 함)
```

- 연결 구간 = **이전 말미 동작의 연속** (새 안무 금지)  
- unique 스토리·엔딩 비트 = **연결 구간이 지난 뒤**에만  
- 편집으로 “연결된 척” 하기보다 **생성에서 이어 그리기**가 정본  

### 0.3 지금 스택의 한계 (정직히)

| | 정본 (0.1) | 현재 grok I2V |
| --- | --- | --- |
| 입력 | **마지막 ~2초 영상** | **마지막 1프레임 이미지** |
| 의미 | 모션 연속 생성 | 정지 사진에서 동작 **재샘플** |
| 결과 | 불필요 연출 없이 자연 연결 | 프롬프트로 “이어라”고만 시킬 수 있음 |

→ 정본 API(영상 꼬리 조건 생성 / video continue)가 오기 전까지는 아래 **근사만** 가능.  
→ 근사를 정본이라고 부르지 않는다.

### 0.4 근사 (도구 제약 시 공통 적용 — 정본 대체 아님)

정본에 **가장 가깝게** 가려면, 순서 고정:

1. **Multi-Signal:** last frame + observed **말미 ~2초 동작**을 packet에 명시  
2. **다음 씬 조인 창:** 그 동작 **그대로 연속만** (새 비트·엔딩 포즈 금지)  
3. **말미 cruise:** 이전 씬도 끊기 직전 2초를 “엔딩 연기”로 망가뜨리지 않음  
4. **조립:** `auto-head-trim` — 고정 2s 폐기로 좋은 조인을 버리지 않음  

이 근사는 “동작을 잘 바꿔서 연결”이 아니라  
**“정본처럼 안 바꾸게 시키는 것”** 이다.

### 0.5 금지 — 정본과 혼동하면 안 되는 것

| 하면 안 되는 것 | 이유 |
| --- | --- |
| **고정 head_trim=2를 “이전 2초 연속”이라고 부름** | 그건 다음 클립 **앞 2초 폐기**일 뿐, 이전 동작 연속이 **아님** |
| 컷 직후 올려다보기·정지·미소 포즈 등 **연결용 연출 추가** | 정본이면 불필요; 근사에서도 **해로움** (C-MOT-CHANGE) |
| still inject / long dissolve로 “이어진 척” | 인계 대체가 아님 ([14](14-multi-signal-handoff-architecture.md)) |
| 정본 불가하니 목표 자체를 폐기 | 목표(§0)는 유지, 구현만 근사 + 도구 업그레이드 대기 |

```text
이전 2초 동작 연속  ≠  다음 클립 head_trim 2초
이전 2초 동작 연속  ≠  경계에 새 동작 넣어 “자연스러워 보이게”
이전 2초 동작 연속  ＝  말미 모션을 끊지 않고 다음 생성이 이어 받음
```

### 0.6 공통 체크 (매 경계)

- [ ] **G1** 목표 문장: “이전 말미 ~2초 동작을 다음이 이어 받는다”를 scene plan에 명시  
- [ ] **G2** 다음 씬 조인 창 프롬프트 = 그 동작 복붙 only (새 안무 없음)  
- [ ] **G3** 이전 씬 말미 ~2초를 엔딩 연기로 망가뜨리지 않음 (cruise 유지 시)  
- [ ] **G4** head_trim을 “2초 연속”이라고 기록·설명하지 않음  
- [ ] **G5** 도구가 video-tail 입력을 지원하면 **정본(모드 B)으로 승격**, 근사 단계 축소

## 1. (참고) 초기 “7–8s unique + 2–3s 브리지” 구조 제안

```text
생성:     |←—— 10s clip N ——→|←—— 10s clip N+1 ——→|
의도:              |← 2–3s overlap 구간 →|
최종 영상: |←— 7–8s unique —→|← shared →|←— 7–8s unique —→|
대본/음원:      7–8s 비트에 맞춤
```

| 구분 | 길이 | 역할 |
| --- | --- | --- |
| I2V 생성 단위 | 10s (도구 한도 6/10) | 모델 입력 제약 |
| **Tail overlap** | **2–3s** | 연결·모션 브리지 |
| **Unique content** | **7–8s** | 대본·내레이션·정보 밀도 |
| 최종 길이 (6씬) | \(6×10 - 5×ov\) | ov=2→**50s**, ov=2.5→**47.5s**, ov=3→**45s** |

“1분 6씬”을 유지하려면:
- 씬 수를 늘거나 (예: 8×10 − 7×2.5 ≈ 62.5s), 또는  
- 생성 길이를 10s로 두고 unique 7.5s × 8비트, 또는  
- 60s 목표를 **콘텐츠 60s**가 아니라 **생성 예산 60s**로 재정의.

## 2. 왜 자연스러워질 수 있는가 (타당성)

| 이점 | 설명 |
| --- | --- |
| **시간 완충** | 컷이 “정보 전환점”이 아니라 “이미 이어지던 2–3초 안”에 놓임 |
| **모션 완충** | B-ACC(말미 감속→초반 가속)를 **오버랩 구간 안에서** 희석 가능 |
| **대본 정합** | 말이 바뀌는 비트 = unique 7–8s, 비주얼 브리지는 대사 없이 가능 |
| **기획 단순** | “10초 만땅 스토리” 대신 “7–8s 비트 + 2–3s 브리지”로 분리 |

즉 **편집·내러티브 설계 레벨의 해법**으로 타당하다.  
lulu/popo에서 실패한 “0.2s still / 1s dissolve”와 달리, **2–3s는 의도된 공유 구간**이다.

## 3. 가능 여부 — 3가지 구현 모드

### 모드 A — 편집 오버랩만 (현재 도구로 **즉시 가능**)

```text
생성: 여전히 이미지→10s I2V + last frame handoff
조립: clip N 의 t=7–10s 와 clip N+1 의 t=0–3s 를 2–3s 겹쳐
      equal-power / hard trim (한 구간만 남김) / 짧은 모션 블렌드
대본: 비트당 7–8s
```

| | |
| --- | --- |
| **가능?** | ✅ 예 (ffmpeg concat/xfade) |
| **조건** | N+1 초반이 N 말미와 **방향·속도·구도**가 어느 정도 맞을 것 (packet 필수) |
| **위험** | 두 클립이 다르면 2–3s 이중노출·고스트 (과거 long dissolve 실패와 유사, 다만 구간이 콘텐츠로 설계되면 완화) |
| **권장 결합** | 단순 crossfade보다 **trim overlap**: N의 7–10s 사용 후 N+1은 **3s부터** 이어 붙이기 (중복 제거) — N+1의 0–3s는 “생성용 워밍업”으로만 쓰고 최종에서 버림 |

**Trim-overlap (권장 변형 A′):**

```text
최종:  [N 0–10] + [N+1 3–10] + [N+2 3–10] + ...
      = N full 10s + 이후 씬마다 7s unique
길이 6씬: 10 + 5×7 = 45s (ov=3) 또는 10+5×8=50s (ov=2)
```

N+1 생성은 여전히 last frame of N으로 시작하되,  
**시청자에게 보이는 첫 프레임은 N+1의 3s 지점** → 모델이 0–3s 동안 frame0 락·모션 안정화할 시간을 줌.

→ **B-VIS / 초반 가속의 시청 노출을 잘라내는 효과.** 실현성 최상.

---

### 모드 B — 생성 시 “꼬리 2–3초”를 입력으로 (조건부 가능)

```text
입력: 이전 클립 마지막 2–3s 영상(또는 그 구간의 키프레임 시퀀스)
출력: 이어서 10s
```

| | |
| --- | --- |
| **가능?** | ⚠️ **현재 grok `image_to_video`는 단일 이미지 시작** — 영상 꼬리 연속 생성 API 없음 |
| **근접** | 말미 1프레임 + packet motion (현행 Multi-Signal) |
| **미래** | video-to-video / motion brush / 다중 프레임 컨디션 지원 시 모드 B 승격 |

---

### 모드 C — 의도적 이중 생성 (비추천 비용)

같은 2–3s를 두 번 생성해 맞추기 → 비용↑·일치 보장 어려움.

## 4. 권장 채택안 (공통 표준)

### Phase 1 (지금 — 정본 근사) — **말미 동작 이어 받기 + auto-head-trim**

정본(§0)이 I2V에서 안 될 때의 **공통 근사** (정본 대체 선언 금지):

```text
1. 목표: 이전 말미 ~2초 동작을 다음이 이어 받음 (§0)
2. 생성: 10s I2V, start=prev handoff, packet에 그 동작 명시
3. 프롬프트: 조인 창 = 그 동작 continue-only (새 안무·엔딩 포즈 금지)
4. 이전 씬 말미: cruise 유지 (엔딩 연기로 2초 망가뜨리지 않음)
5. 조립: concat --auto-head-trim (head_trim=2를 “2초 연속”이라 부르지 않음)
6. 기록: join_score / auto_head_trim JSON
```

| 장점 | 단점 |
| --- | --- |
| 최량 조인 프레임을 버리지 않음 | frame0 불량 시 trim>0 필요 |
| 동작 변경 티 1순위 감소 | 조인 창에 스토리 밀도↓ |
| CLI 구현됨 | max trim 이상 드리프트는 재생성 |

### Phase 1b (선택) — 고정 A′ head_trim

frame0 ΔY/MAE 실패·초반 가속이 심할 때만:

```text
--head-trim 1~2  또는  --auto-head-trim --head-trim-min 1
대본 unique 를 7–8s 로 맞추면 길이 예측 가능
```

### Phase 2 — 대본/음원 계약 (trim 적응형)

```yaml
scene_duration_gen: 10
assembly:
  mode: auto_head_trim      # 공통 기본
  head_trim_max_sec: 2.0
  # head_trim_sec: 2.0      # 고정 모드는 예외
content:
  join_window_sec: 8.0      # continue-only
  end_beat_last_sec: 2.0
audio:
  plan_for_join_window: true  # 조인 창 무대사 권장
```

### Phase 3 — 생성기 업그레이드 시 **정본 승격** (모드 B)

말미 ~2초 영상 입력이 가능해지면:

```text
입력 = prev clip tail ~2s → 출력 = continue 10s
→ §0 정본 그대로 구현
→ 근사 단계(강제 continue 문장·trim 고민) 축소
```

## 8. 권고 기본값 (공통)

```yaml
generation:
  scene_seconds: 10
assembly:
  mode: auto_head_trim     # 공통 기본 (head_trim | timeline_overlap | hard_cut 은 예외)
  head_trim_max_sec: 2.0
  head_trim_min_sec: 0.0
  head_trim_step_sec: 0.5
content:
  join_window_sec: 8.0     # continue-only
  end_beat_policy: final_scene_tail_only
audio:
  plan_for_join_window: true
```

**1차 권장 모드: `auto_head_trim` + 조인 창 continue-only.**  
고정 `head_trim` (A′)는 frame0 불량 시 보조.
